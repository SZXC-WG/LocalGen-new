## 格式状态

| 文件 | 用途 | 当前 v6 状态 |
| --- | --- | --- |
| `.lgmp` | v6 原生地图 | 地图编辑器可读写；本地对局与模拟器可读取 |
| `.lg` | v5 旧地图 | 地图编辑器可读写；不会保留元数据；本地对局不直接列出 |
| 官方 `.json` | generals.io 地图交换 | 地图编辑器可读取/导入；不能导出 JSON；不可直接开局 |
| `.lgr` | 普通回放 | 上游参考中有定义，但当前 v6 没有读写器 |
| `.lgra` | 高级回放 | 上游参考中有定义，但当前 v6 没有读写器 |
| `settings.lgsts` | v5 设置 | 仅为旧版参考 |
| `settings.json` | v6 设置 | 文档中的文件名；当前应用不会持久化设置 |

## v6 原生 `.lgmp`

`.lgmp` 是 Qt 数据流二进制格式，并不是 JSON。它保存：

- 格式魔数；
- 标题、作者、创建日期时间与描述；
- 地图宽度与高度；
- 压缩格子记录，其中包含格子类型、全局可见的“点亮”位，以及兵力或出生队伍值。

地图编辑器支持 1×1 至 100×100。用于对局的地图必须为所有参与者提供足够的出生格或空白平地。

## 旧版 `.lg`

当前编辑器可以读取和写入 v5 棋盘编码。打开时会根据文件生成标题与创建时间，但保存 `.lg` 会丢弃 v6 元数据。新地图建议使用 `.lgmp`。

## 官方地图 JSON

编辑器接受包含有效 `width`、`height`、`map`、`title` 与 `created_at` 数据的官方地图对象，也可包含作者和描述。它还可以按名称从 generals.io 公开地图 API 获取同类数据。导入后，如需在 LocalGen 中使用，请另存一份本地 `.lgmp`。

## 让地图出现在对局中

把有效 `.lgmp` 放入 `LocalGen-new` 可执行文件旁的 `maps/` 目录。本地对局窗口打开时会扫描该目录，并优先使用地图元数据标题作为显示名称。

回放扩展名与设置文件名仍是项目术语的一部分，但在相应读写器实现之前，不应把它们描述成可用的 v6 功能。
